# 下一轮分发顺序

这轮不再扩实现面，先复核现行入口，再整理提交前验证和分组建议，最后做一次很薄的交叉复查。

## 第一阶段

并行启动：

- `Prompt-1`
- `Prompt-2`

目标：

- 复核用户当前真实可达的建服/接入路径有没有漏传或桥层不一致
- 按当前工作区实际改动整理提交前验证口径和建议的 commit 分组

## 第二阶段

在 `.cache1` 和 `.cache2` 回来后，再启动：

- `Prompt-3`

目标：

- 只处理前两条线之间的小冲突和文件归属问题
- 判断是否已经可以进入分类提交阶段

## 说明

- 这轮至少保留 2 个并行 Agent。
- `Prompt-1` 不要改运行时主线，`Prompt-2` 不要主动扩功能实现。
- `.cacheN` 只作为回传，不要 commit。
