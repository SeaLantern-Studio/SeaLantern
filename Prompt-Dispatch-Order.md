# 下一轮分发顺序

这轮先把“实例配置写入”和“运行时真正按它读取”对齐，再收入口透传，最后做一次薄复核。

## 第一阶段

并行启动：

- `Prompt-1`
- `Prompt-2`

目标：

- 让本地启动链、Docker 链、真实启动详情都优先读实例配置
- 把建服页和接入页所有可达入口的 `cpu_policy + jvm_preset + jvm_args` 透传补齐

## 第二阶段

在 `.cache1` 和 `.cache2` 回来后，再启动：

- `Prompt-3`

目标：

- 复核保存后刷新是否能看到最新真实启动详情
- 只补和本轮主线直接相关的小断口

## 说明

- 这轮至少保留 2 个并行 Agent。
- `Prompt-1` 不要编辑前端页面，`Prompt-2` 不要改运行时读取主线。
- `.cacheN` 只作为回传，不要 commit。
