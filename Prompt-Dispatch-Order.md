# 下一轮分发顺序

这轮不要 4 个 Agent 同时启动，按两阶段走，保证至少 2 个并行。

## 第一阶段

并行启动：

- `Prompt-1`
- `Prompt-2`

原因：

- `Prompt-1` 负责把共享 schema、DTO、默认值和兼容规则拍板。
- `Prompt-2` 负责本地运行时补丁级方案，虽然会参考 schema，但本地链路结论已经足够清楚，可以先并行推进。

第一阶段完成条件：

- `.cache1` 返回最终合同版本
- `.cache2` 返回本地运行时补丁级方案

## 第二阶段

在读完 `.cache1` 后，再并行启动：

- `Prompt-3`
- `Prompt-4`

原因：

- Docker 运行时和前端字段展示都更依赖最终 schema 拍板结果。
- 把它们放到第二阶段，可以减少因为字段名、默认值或范围变化而返工。

第二阶段输入要求：

- 启动 `Prompt-3` 和 `Prompt-4` 前，把 `.cache1` 一并提供给对应 Agent。

## 这样拆的目的

- 保证每一阶段至少 2 个 Agent 并行，不会串行得太慢。
- 避免 4 个 Agent 同时围绕一个未定的 schema 发散，导致后续还要重新收敛。
- 先定共享合同，再放依赖它更重的 Docker 和前端方向，整体返工最少。
